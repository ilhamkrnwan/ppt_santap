From Concept to Reality

Latar belakang dikembangkannya Santap adalah masalah nyata pada operasional restoran manual — seperti antrean panjang di kasir, kesalahan pencatatan pesanan oleh pramusaji (human error), dan rekapitulasi data keuangan yang rawan manipulasi serta tidak terintegrasi antar bagian.

Untuk itu, kami membangun ekosistem POS terintegrasi. Mobile POS (Flutter) untuk kasir di tablet, Dashboard Admin (Filament) untuk owner memantau laporan, dan Website Customer (Nuxt 4) di mana pelanggan cukup men-scan QR Code di meja untuk langsung memesan. Seluruh komponen tersebut dihubungkan melalui Backend REST API (Laravel 13) yang menjadi pusat autentikasi, transaksi, dan sinkronisasi real-time.
